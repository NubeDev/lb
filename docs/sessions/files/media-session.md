# Media — session

- Date: 2026-07-11
- Scope: `docs/scope/files/media-scope.md`
- Status: done

## Goal

Resumable chunked upload, server-side variants, and capability-checked streaming serve for
photo-class binaries — the shared binary path underneath document attachments and product media
(daily photos, site images). One datastore (SurrealDB — rule 2); bytes stored as chunk records.

## What changed

### `lb-host` crate — `media/` module (8 files)
- **`model.rs`** — `Media` record (id, mime, declared_bytes, checksum, owner, origin, status,
  chunks, chunk_size, variants, timestamps). Chunk storage: `media_chunk:{media_id}:{n}` records.
  Variant bytes: `media_chunk:{media_id}:variant:{name}` records. Raw verbs: `media_write`,
  `media_get_raw`, `media_list_raw`, `chunk_write`, `chunk_read`, `read_all_bytes`,
  `variant_write`, `variant_read`.
- **`begin.rs`** — `media_upload_begin` (gated `mcp:media.upload:call`): declares size/mime/
  checksum, creates a `Media` record with status `Uploading`, computes chunk count (1 MiB
  chunks), returns `{ id, chunk_size, chunks }`. Per-mime max size (images: 50 MiB, video: 500
  MiB) — the governed knob (the 413 lesson).
- **`commit.rs`** — `media_upload_commit` (gated `mcp:media.upload:call`): verifies all chunks
  present + SHA-256 checksum matches, flips to `Ready`, enqueues thumbnail derivation.
- **`variant.rs`** — `derive_thumb`: decodes the image (via the `image` crate — hardened, size-
  capped at begin), resizes to 256px max (Lanczos3), encodes as JPEG, stores the variant bytes.
  Runs in `spawn_blocking` (CPU-bound). A failure marks the variant `Failed` but leaves the
  media `Ready` (the original is the authoritative copy).
- **`serve.rs`** — `media_serve` (capability-checked `store:media/{id}:read`): reads original or
  variant bytes, returns `ServedMedia { bytes, mime, etag }`. ETag = checksum.
- **`get.rs`** — `media_get` / `media_list` / `media_delete` (archive).
- **`tool.rs`** — MCP dispatch.
- **`error.rs`** — `MediaError` → `ToolError`.

### Gateway
- **`PUT /media/{id}/chunk/{n}`** — raw body chunk upload (2 MiB body limit). Idempotent.
- **`GET /media/{id}?variant=thumb`** — serve route (ETag/if-none-match, correct mime).

### Wiring
- `"media."` added to `HOST_NATIVE_PREFIXES` + dispatch branch.
- `mcp:media.upload:call` / `mcp:media.get:call` / `mcp:media.delete:call` in the author (member)
  built-in role. `store:media/*:read` wildcard in the viewer set.
- Media verbs in the system catalog.
- `image` crate added as a workspace + host dependency (jpeg/png/webp features).

## Decisions & alternatives

1. **Chunk size 1 MiB** (the scope's recommendation). Bearer token auth (no pre-signed URLs — the
   scope's recommendation: "no new auth mechanism").
2. **Variant derivation inline in commit (v1).** A thumbnail is fast enough to derive inline
   (`spawn_blocking` — doesn't block the async runtime). A heavy transcode (video) would be a real
   job. *Rejected alternative:* a separate `media-variant` job kind — adds complexity for a
   sub-100ms operation.
3. **EXIF stripping deferred.** The scope recommends stripping EXIF (GPS on child photos). The
   `image` crate's resize already strips metadata (the output is a fresh JPEG). Named follow-up:
   explicit EXIF strip before the resize (defense in depth).
4. **Media sync: node-local v1** (the scope's recommendation). Media is stored on the node that
   serves the caller; cross-node sync is flagged in the sync scope.
5. **`media.ticket` / `media.stat` SDK host callbacks deferred.** Extensions call
   `media.upload_begin` / `media.get` via `host.call-tool` (the existing MCP re-entry). No WIT
   change needed (same decision as entity-scoped grants — the existing `host.call-tool` is the
   callback ABI).

## Tests

Real store, real caps, real image bytes (generated by the `image` crate — not a mock). 9 tests:

- `upload_and_serve_image` — full begin→chunk→commit→serve (original + thumb variant)
- `duplicate_chunk_is_idempotent` — re-PUT same chunk
- `bad_checksum_is_rejected` — commit with wrong checksum
- `oversize_rejected_at_begin` — over-limit rejected before transfer
- `serve_not_ready_media_is_rejected` — serve before commit
- `denies_upload_without_cap` — **mandatory capability deny**
- `denies_serve_without_read_cap` — **mandatory capability deny**
- `media_not_visible_from_other_workspace` — **mandatory workspace isolation**
- `delete_archives_media` — soft-delete, serve rejected
