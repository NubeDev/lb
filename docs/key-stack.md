# Key stack

Status: draft stack map. This lists the main libraries, frameworks, and internal
crates expected by the architecture scope, plus where each is used across edge
and cloud roles.

| Area | Library / tool | Edge use | Cloud use | Notes |
| --- | --- | --- | --- | --- |
| Datastore | [**SurrealDB**](https://github.com/surrealdb/surrealdb) | Embedded local store for node-local data, cached workspace data, files/buckets, jobs, inbox/outbox, tags, docs, and extension metadata. | Embedded/server-side store for hosted workspaces, identity/team/workspace authority, registry metadata, files/buckets, jobs, inbox/outbox, docs, and audit records. | The only persistence layer. State lives here; messages do not. |
| Message bus | [**Zenoh**](https://github.com/eclipse-zenoh/zenoh) | Peer mode. Connects to the hub, publishes/subscribes to workspace key expressions, supports local/LAN peers, presence, and routed MCP calls. | Router mode. Accepts edge peers, routes workspace traffic, central agent traffic, gateway streams, and extension queryables. | Motion lives here. Must-deliver work still goes through outbox. |
| Host / kernel | `host` internal crate | The node kernel on each device: mediates every capability check, runs the MCP server, supervises extensions, wires store/bus/runtime together. | Same kernel, hosting many workspaces. | Nothing reaches the store, bus, secrets, or a peer except through the host. |
| Host introspection | [`if-addrs`](https://github.com/messense/if-addrs) + [`hostname`](https://github.com/svartalf/hostname) + [`chrono`](https://github.com/chronotope/chrono) + [`chrono-tz`](https://github.com/chronotope/chrono-tz) + [`iana-time-zone`](https://github.com/strawlab/iana-time-zone) | Built-in `host.*` MCP verbs read node-local interface, hostname, time, timezone, and filesystem metadata facts without shelling out. | Same verbs, same DTO shape, reporting the cloud node's local facts when the call runs there. | Added for `host-tools`: cross-platform crates checked by `cargo check -p lb-host` under the repo's zig/no-system-cc toolchain. Filesystem uses `std::fs`; `host.net.reach` uses `TcpStream::connect_timeout`. |
| Git / CLI tools | `lb-gh` internal crate (ported from `lazybones-gh`) over the `gh` + `git` CLIs | Drives the `git.*` MCP verbs (`git.commit_push`/`git.status`) for node-local repo ops — AI-free periodic auto-commit-and-push fired by a `reminder` → `lb-jobs` job. | Same crate + verbs on a hosted node. | The sanctioned **true-external** seam (CLAUDE §9): shells out to the user's already-authenticated `gh`/`git`, so there is **no token handling in-process**. Folder-of-verbs like `host.*` — more CLI-backed tool families add a folder, not a new mechanism. `scope/git-sync/`. |
| MCP/tool layer | [**rmcp**](https://github.com/modelcontextprotocol/rust-sdk) | Local MCP server exposes tools from local extensions and routes calls to cloud-hosted tools/agents through Zenoh. | Hub MCP server exposes cloud extensions, registry/admin tools, central AI agents, and AI gateway tools. | Universal contract for UI, AI agents, and extensions. |
| Extension runtime | [**Wasmtime**](https://github.com/bytecodealliance/wasmtime) | Runs sandboxed WASM extensions on user devices; useful for local-only tools, offline workflows, and cached extensions. | Runs cloud-only or either-placement WASM extensions, including workflow, registry, admin, and automation extensions. | Tier 1 default runtime. Capability-gated and resource-limited. |
| Extension ABI | [**WASI 0.2**](https://github.com/WebAssembly/WASI) / [Component Model + WIT](https://github.com/WebAssembly/component-model) | Shared extension interface for edge-hosted WASM components. | Shared extension interface for cloud-hosted WASM components. | Stable plugin boundary; version deliberately because extension authors depend on it. |
| Native extension tier | Native sidecars | Optional local sidecars for filesystem, IDE, language server, GPU, or device-specific access. | Optional supervised sidecars for heavy compute, language servers, or provider adapters. | Tier 2 escape hatch; never dynamically linked Rust plugins. |
| Extension loading | `ext-loader` internal crate | Pulls, verifies, caches, and starts extension artifacts for edge use, including offline cache. | Loads registry-hosted artifacts, manages cloud placement, hot reload, and rollback. | Internal library wrapping registry, runtime, and placement rules. |
| Extension author SDK | SDK crate + `@lazybones/ext-sdk` | Rust/WIT bindings for backend extensions; frontend SDK for local/desktop UI extension slots. | Same backend SDK; frontend SDK for cloud-served extension UIs. | Public API surface for extension authors. |
| Extension registry | `lb-registry` crate + host `registry` service (registry host = role) | Client pulls signed artifacts, **verifies** them (`verify_artifact`), caches them as SurrealDB records, and runs offline once cached; rollback = install a prior version. | Hosts catalog, metadata, signed artifacts, private/public visibility, and install/update MCP tools (real HTTP source = S7 follow-up). | **Signing resolved (S7):** Ed25519 over a SHA-256 digest binding manifest+wasm, reusing the `lb_auth` `ed25519-dalek` stack — no second crypto library. Bytes-as-record until the `DEFINE BUCKET` config swap. |
| Auth/OIDC | OIDC + JWT | Verifies tokens offline using hub public keys; operates on cached identity and workspace claims. | Issues tokens, manages OIDC login, global identity, workspace membership, and RBAC authority. | Library choice still TBD. Identity and token issuance; capability enforcement is `caps`. |
| Capabilities | `caps` internal crate | Enforces workspace isolation first, then capability checks, on every local store/bus/MCP/secret access. | Same enforcement across all hosted workspaces. | The actual core product. One scope model projecting onto store + bus + MCP, kept in sync. |
| Secrets | [`keyring`](https://github.com/open-source-cooperative/keyring-rs) + envelope encryption | OS keychain on desktop where available; encrypted workspace/extension secrets in local store. | Env/KMS-backed master key wraps workspace keys and extension secrets. | Extensions request secrets through host-mediated capabilities. |
| Jobs | SurrealDB-native queue or [`apalis`](https://github.com/apalis-dev/apalis) backend | Local durable jobs, offline workflows, sync relays, and edge-hosted workflow sessions. | Remote workflow sessions, central coding jobs, scheduled jobs, retries, cron, and cloud workers. | Exact queue implementation is still open. Jobs are records in SurrealDB. |
| Inbox/outbox | Internal crates | User attention items, local approvals, sync queueing, and durable outbound work while offline. | Workspace/team inbox, approval routing, webhook/event intake, external delivery, and sync fanout. | Inbox is attention. Outbox is durable external effect. |
| Tags/search | SurrealDB graph + full-text/vector features | Local tagging/search over docs, files, inbox items, jobs, messages, and cached workspace data. | Workspace-wide tagging/search over shared docs, messages, inbox, extensions, jobs, and audit records. | Key:value tags with graph edges and faceted search. |
| Files/docs | SurrealDB buckets | Local disk-backed buckets for offline docs, files, generated outputs, and cached shared assets. | S3-compatible backing for scale, shared document store, registry artifacts, and generated outputs. | Blob strategy at scale is still an open decision. |
| AI agents / workflow | Workspace-scoped actors, built as extensions | Local agents for offline/local-only tasks; can invoke the central agent remotely over MCP. | Hosts central workspace-scoped agents and durable workflow sessions (jobs). | Not core; composed from primitives. Agents call the AI gateway for model access and **own the tool-call loop**. |
| Shared AI gateway | Swappable model-access sidecar (Tier-2) behind a stable contract | Optional local-provider mode; can enforce local-only workflows and refuse remote routing. | Shared model/provider gateway with workspace secrets, routing, streaming, quotas, retention policy, and audit. | Not an agent and does **not** run the tool-call loop; agents/jobs call through it. Implementation is swappable (custom service, SpiceAI, LiteLLM-style proxy, local-model wrapper). |
| Agent run stream / client protocols | `RunEvent` vocabulary + per-protocol encoders (`role/acp` = [Agent Client Protocol](https://agentclientprotocol.com) stdio role; AI-SDK v6 next) | Solo node serves the same stream + ACP adapter locally; an editor drives the local agent over stdio. | Hub emits the run stream; the ACP adapter (and a future web AI-SDK encoder) run as roles beside the gateway. | One internal event vocabulary the loop emits; every wire format is a thin encoder over it (ACP, AI-SDK, the gateway SSE feed). MCP stays the internal contract; these are edge translations. Awaken-reviewed; its `trait Plugin` loop-hook framework rejected. `scope/agent-run/`. |
| External agent runtime (optional) | [**Agent Client Protocol** Rust SDK](https://github.com/agentclientprotocol/rust-sdk) (`agent-client-protocol` + `-tokio` + `-rmcp`); default agent [**VT Code**](https://github.com/vinhnx/vtcode) (MIT), alternate [**dirge**](https://github.com/dirge-code/dirge) (GPL-3.0) | Drives a locally-installed ACP agent for local/offline coding/data tasks; same seam, the gateway resolves a local provider. | Hub drives the agent as a sandboxed subprocess; tools = our caps-checked MCP surface, models = our gateway. | **Compile-time feature** (`external-agent`, off by default) behind a host-owned `AgentRuntime` trait — the default runtime is the in-house loop. We **drive** the agent over ACP (we are the *client*), never embed/fork it; the whole ACP registry is pluggable by config. `scope/external-agent/`. |
| AI/data plane | [**SpiceAI**](https://github.com/spiceai/spiceai) optional | Usually off for minimal/Pi profile; possible local provider or retrieval adapter when enabled. | Optional full-profile adapter for federated SQL, vector/text search, and LLM gateway behavior. | One possible implementation behind the AI gateway contract, not a second gateway. |
| Rules engine | `lb-rules` crate + [**rhai**](https://github.com/rhaiscript/rhai) (sandboxed scripting) | In-process rule/processing engine: a governed `rhai` cage + a lazy `Grid` + a verb library; reads platform data via `data.query`/`series.*`, external via `federation.query`, `ai.*` via the gateway, `emit`/`alert` via inbox/outbox. | Same crate; hosts saved rules + scheduled flows for hosted workspaces. | Ported from `rubix-cube` onto the lazybones chokepoints. Library, not a service; the cage = capability-first, in-process. No DataFusion in core. `scope/rules/`. |
| Rule DAG (flows) | `lb-flows` model + host flow engine over the **`lb-jobs`** queue | Local flow runs; each node is a durable job; cron via the S6 reactor, event via `bus.watch`, plus inject/boot triggers. | Same; hosted flows across workspaces, resumable after restart. | The `rubix-cube` workflow DAG (`validate`/`indegrees`/`dependents`/binding resolver) lifted into **`flows`** — the one DAG engine (the earlier `chains` engine was **retired**, `flows/chains-retirement-scope.md`). `JobQueue`/run-store/cron seams swapped for `lb-jobs` + SurrealDB + the reactor. `scope/flows/`. |
| Flow data/JSON nodes (parse crates) | [`csv`](https://github.com/BurntSushi/rust-csv) + [`quick-xml`](https://github.com/tafia/quick-xml) + [`serde_yaml`](https://github.com/dtolnay/serde-yaml) + [`base64`](https://github.com/marshallpierce/rust-base64) | The four minimal, pure-Rust readers/writers behind the `csv`/`xml`/`yaml`/`base64` built-in flow nodes (data-nodes scope). Parse fails the node on malformed input (`json`-node parity). Pulled into **`lb-flows`** only — the ops are pure functions of a JSON value (`ops::parse`), unit-tested in-crate. | Same crates; a node runs identically edge or cloud (pure, no I/O). | Deliberately no heavy templating engine (`template` node is a hand-rolled mustache-lite over serde values). BurntSushi CSV + tafia quick-xml (no C++), dtolnay serde_yaml, marshallpierce base64 — all vetted, dependency-light. `scope/flows/data-nodes-scope.md`. |
| Reminders cron | [`croner`](https://github.com/Hexagon/croner-rust) | Parses the reminder record's 5-field cron `schedule` and computes "next after T" via `find_next_occurrence(after, inclusive=false)` on the **injected logical clock** — never the wall clock, so firing is deterministic on edge nodes. | Same crate; the hosted reactor advances `next_attempt_ts` identically. | The one new mechanical piece of the reminders slice (`crates/reminders/src/next_after.rs`). Vetted Vixie-cron-compatible parsing/next-time; not hand-rolled (multi-value fields / month-day rollover). `scope/reminders/`. |
| Reminders cron builder (UI) | [`react-js-cron`](https://github.com/xrutayisire/react-js-cron) | The visual cron authoring component (most users don't read cron): reads/writes a standard 5-field cron string losslessly. antd-based — wrapped in **one** component (`CronBuilder.tsx`) and **restyled** to the shell's Tailwind/shadcn tokens via a locally-scoped antd `ConfigProvider`. | Same component served in the browser UI. | antd is **not** pulled into the global theme — it is scoped to this subtree only. Pinned to `react-js-cron ^6.0.2`. `scope/reminders/`. |
| External datasources | `federation` native (Tier-2) extension + [**DataFusion**](https://github.com/apache/datafusion) + [`datafusion-table-providers`](https://github.com/datafusion-contrib/datafusion-table-providers) | Connects to MySQL/PostgreSQL/TimescaleDB/… under `net:*` + a mediated secret; read-first `federation.query`, workspace-pinned; `federation.mirror` pulls ranges into the series plane. | Same extension, hosted; mirror jobs cache external ranges for dashboards. | Embeds DataFusion **as a library** (the engine Spice/`rubix-cube` build on), never the Spice/Cube product as a service. SurrealDB stays authority; external DBs are federated sources, never a second store. `scope/datasources/`. |
| Control Engine bridge | `control-engine` native (Tier-2) extension + `ce-client-rust` (the `ControlEngine` trait: REST commands/reads + binary-WS COV) + vendored `@nube/ce-wiresheet` (React Flow editor, `packages/ce-wiresheet`) | Binds a `localhost` CE over REST/WS and serves `ce.*` MCP tools; on an appliance edge node it owns the local CE socket (`net:tcp:127.0.0.1:<port>`) and publishes COV via the generic extension-watch primitive (`ce.watch`). | Same extension; a `ce.*` call for a remote appliance is a **routed MCP hop over Zenoh** to the node that owns that CE. | **100% an extension — the core stays CE-ignorant (CI-enforced).** No CE-on-Zenoh codec, no raw proxy (README §3 rules 1/3/7). `ce-client-rust` (git dep) and `@nube/ce-wiresheet` (`workspace:*`, transport re-pointed onto the MCP bridge) are upstream Nube libraries updated **with approval first**. Scope co-located: `rust/extensions/control-engine/docs/`; the one generic core addition it implies is `scope/extensions/extension-watch-scope.md`. |
| Preferences / localization | `lb-prefs` crate + [**icu4x**](https://github.com/unicode-org/icu4x) (CLDR i18n: date/number/list formatting, tz, ICU MessageFormat) + [**uom**](https://github.com/iliekturtles/uom) (typed unit conversion) | Resolves the local principal's prefs (language/tz/date/units) and runs all formatting/conversion **offline** — compiled-in CLDR, no round-trip. | Same crate; localizes server-generated content (emails, notifications, inbox renders) in the recipient's language; holds workspace-default prefs + override catalogs. | Canonical data stored (UTC, SI/base units, neutral codes); presentation resolved at the boundary as `prefs.*`/`format.*`/`convert.*` MCP tools so thin clients don't re-implement it. Compile in only enabled locales (en/es) for the Pi profile. |
| Dashboard data resolve / transforms | `lb-viz` crate (pure transform lib, Grafana's transformer set) | Resolves a panel's `viz.query`: dispatches `sources[]` targets under `caller ∩ grant` then applies the transformation pipeline (reduce/organize/filter/groupBy/joinByField/calculateField/…), returning **canonical** frames. | Same crate; resolves panels for the cloud gateway and any thin client (React Native, email, webhook) identically. | The data-resolve twin of `lb-prefs`: one pure lib + an MCP verb (`viz.query`/`viz.stream`) so no client re-implements the pipeline. Not a new compute plane — pure functions over already-fetched, gated, bounded rows; heavy aggregation pushes to the query/`lb-jobs`. `scope/frontend/dashboard/viz/transformations-scope.md`. |
| Generative UI (agent-authored widgets) | `@nube/genui` internal package (A2UI-shaped IR + catalog renderer) + [`@openuidev/lang-core`](https://github.com/thesysdev/openui) (MIT, pure OpenUI-Lang parser) | Renders `view:"genui"` cells from the persisted spec + live `sources[]` bindings — no model in the render path, works offline once authored. | Same package; the hosted agent streams the authoring preview over the run-stream SSE. | IR follows [A2UI v0.9](https://a2ui.org/specification/v0.9-a2ui/) message patterns (flat id maps, JSON-Pointer data model, typed actions) **without** depending on Google's renderer (pre-1.0 churn); OpenUI Lang is the default token-cheap emission format via one adapter each. Neither library's styled components are vendored (CSS scoping rules). `scope/genui/`. |
| Frontend shell | [**React**](https://github.com/facebook/react) + [TypeScript](https://github.com/microsoft/TypeScript) | Tauri desktop shell renders the same React app against the local node. | Browser app served by the cloud gateway. | One UI codebase. |
| Styling/components | [**Tailwind CSS**](https://github.com/tailwindlabs/tailwindcss) + [shadcn/ui](https://github.com/shadcn-ui/ui) + [Radix UI](https://github.com/radix-ui/primitives) | Desktop/local UI components and extension UI tokens. | Browser/cloud UI components and extension UI tokens. | Keep extension UIs visually native. |
| Desktop wrapper | [**Tauri v2**](https://github.com/tauri-apps/tauri) | Primary desktop shell for Windows/macOS/Linux; talks to local host directly. | Not used as the cloud runtime. | Cloud serves web UI; edge can use Tauri. |
| Frontend extension loading | [**Module Federation Vite**](https://github.com/module-federation/vite) | Trusted first-party extension UIs can be loaded into the local/desktop app when served by the node. | Cloud hosts and serves trusted extension remotes. | Untrusted UIs should use Web Components or iframes. |
| Untrusted frontend isolation | Web Components / sandboxed iframes | Isolates third-party UI surfaces from the host origin where needed. | Same isolation model for cloud-served browser UI. | Module Federation is full-trust; iframe/Web Component path is for untrusted surfaces. |
| Gateway/API | SSE + HTTP POST | Edge UI talks directly to local host; generally no SSE needed for desktop-local path. | Browser clients receive server-to-browser updates over SSE and send commands over HTTP POST. | Rust web framework is still TBD unless the coding scope chooses one. |
| Bootstrap UI | Minimal embedded UI | Not usually needed on normal edge nodes. | First-run cloud setup; mints first super-admin token, then locks. | Privileged cloud-only surface. |
| Sync | Internal `sync` crate | Change feeds to outbox, publishes queued writes, receives hub updates, supports offline replay. | Applies edge-originated writes, routes workspace updates, acts as shared-data authority. | Authority-partitioned sync, not general multi-master replication. |
| Real-time co-editing | [Automerge](https://github.com/automerge/automerge) / [Yjs](https://github.com/yjs/yjs) (CRDT) | Local CRDT state for collaborative docs inside an extension. | Relays/merges CRDT updates between collaborators. | **Per-extension concern, not a platform feature** (§6.8). Distinct from authority-partitioned sync. |
| Node binary / roles | `node` binary | Wires the core crates and reads config to enable the edge role(s); selects local placement and offline behavior. | Same binary; config enables router, multi-workspace hosting, registry host, gateway. | One binary, all roles. Edge vs cloud is config, never a code branch. |
| Operator CLI | `lb-cli` crate (`lb` binary) + [**clap**](https://github.com/clap-rs/clap) (derive) + [`reqwest`](https://github.com/seanmonstar/reqwest) + [`tabled`](https://github.com/zhiburt/tabled) | `lb local …` embeds the host (`Node::boot`) and calls `call_tool` in-process — the offline/solo posture, no daemon. | `lb …` (remote) talks to the gateway over the same SSE/HTTP path the browser uses; `POST /mcp/call` is the universal verb. | A fourth client (besides browser/Tauri/mobile) — holds **no authority of its own**; it is only ever as authorized as the token it presents, denied whenever the server denies. Adds no verbs/caps/tables; the named first consumer of API keys (`scope/cli/`). |
| Observability | [`tracing`](https://github.com/tokio-rs/tracing) + `tracing-subscriber` + [`tracing-opentelemetry`](https://github.com/tokio-rs/tracing-opentelemetry) (OTLP) | Emits spans/logs/metrics; sink = stderr/rolling file; buffers when a collector is unreachable, exports on reconnect. | Same emission; sink = OTLP exporter to a collector (Tempo/Loki/Prometheus). `trace_id` propagates across the routed Zenoh hop + into jobs/outbox. | Platform **emits**; collection/dashboards are external. Secret-safe via `Secret<T>` redaction. Not the audit log (sampled, droppable). `scope/observability/`. |
| Audit ledger | Internal `lb-audit` crate (SurrealDB rows + SHA-256 hash chain, reuses the `lb_auth` crypto stack) | Local, intact, verifiable per-(ws,node) chain offline; syncs append-only to the hub. | Workspace ledgers + reserved `_lb_audit_system` for cross-tenant admin; the union + chain verification. | Immutable/append-only (no caller write verb); records refs/digests, never secrets. Generalizes §6.14's model-call audit. `scope/audit/`. |
| Undo / reversible commands | Internal `lb-undo` crate (before-image journal + store-managed `rev`, over the `write_tx` seam) | Per-(ws,actor) bounded undo/redo stacks (immutable events + materialized cursor); restore is a **conditional** write enforced at the authoritative node. | Same mechanism; admin `undo.any` override (audited). | **Reverse state, compensate motion** — host *derives* irreversibility from runtime transaction taint (reaching the outbox). Not collaborative/OT undo (a per-extension CRDT concern, §6.8). `scope/undo/`. |

## Placement summary

| Capability | Edge | Cloud hub |
| --- | --- | --- |
| Local/offline work | Primary role | Not applicable |
| Capability enforcement | Host-mediated on the local node, workspace-first | Same, across all hosted workspaces |
| Workspace authority | Cached shared data plus local data | Authority for shared identity, teams, workspace data, registry, and hosted workflows |
| Central AI agent | Can invoke remotely; may run local agents too | Hosts workspace-scoped central agents |
| Shared AI gateway | Optional local-provider mode | Primary shared provider gateway |
| Extension runtime | Runs local/either extensions | Runs cloud/either extensions |
| UI | Tauri desktop and possible local web | Browser UI, bootstrap UI, admin surfaces |
