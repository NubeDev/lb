# Product

## Register

product

## Users

Workspace operators, admins, and extension builders use Lazybones while inspecting local-first workspace state, routing work through extensions, and checking capability-scoped data. They are usually in a focused maintenance or debugging flow and need dense, trustworthy interfaces that make the current workspace and permission boundary obvious.

## Product Purpose

Lazybones is a reusable edge/cloud platform for extension-driven workspace software. The UI exposes shared core surfaces for data, messages, approvals, members, extensions, and admin controls so teams can operate the platform without dropping into raw infrastructure tooling. Success looks like confident inspection, fast triage, and no ambiguity about what is read-only, mutable, local, or workspace-scoped.

## Brand Personality

Quiet, capable, and exact. The product should feel like a serious control surface for people who understand systems, with just enough warmth to keep repeated operational work from feeling brittle.

## Anti-references

Avoid marketing-site gloss, decorative dashboard theatrics, fake glass panels, oversized hero typography, and database tools that hide raw values behind cute abstractions. Avoid low-contrast terminal cosplay, unbounded JSON strings, and table layouts that make admins fight horizontal scanning.

## Design Principles

- Preserve the workspace wall: always make scope and read/write affordances legible.
- Prefer inspection over spectacle: dense information is welcome when hierarchy and scanning are clear.
- Make raw data humane: keep values faithful, but format and color them so structure is readable.
- Use standard product affordances: tabs, tables, sidebars, and graph canvases should behave like familiar tools.
- Keep capability boundaries honest: privileged surfaces should look controlled, not playful.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Do not rely on color alone for data type or state meaning; pair semantic color with labels, symbols, or structure where it matters. Respect reduced-motion preferences and keep keyboard focus visible on dense controls.
